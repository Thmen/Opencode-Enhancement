# 离线文档面包屑导航设计

## 背景

当前离线 Markdown 文档已经可以通过根目录 `README.md` 进入各个章节，但在进入具体页面后，缺少像网页版那样的“返回首页/返回上级分类”导航。以 `docs/zh/opencode-bk/使用/07-分享.md` 这类页面为例，用户打开后只能依赖编辑器的返回历史，不够直观，也不利于把离线文档当作静态文档站来浏览。

本次改动的目标是在不改变正文语义的前提下，为生成后的离线文档补充稳定、可点击的层级导航。

## 目标

- 为所有离线正文页增加页首面包屑导航
- 根目录 `README.md` 继续作为文档首页
- 为每个分类目录额外生成一个 `README.md` 作为分类首页
- 中文与英文离线文档共用同一套导航生成逻辑
- 不依赖 Markdown 渲染器对中文锚点的兼容性

## 非目标

- 不改写页面正文内容
- 不加入页尾导航、浮动目录或复杂 UI
- 不改变现有外链和页内锚点的处理规则
- 不对历史备份目录 `docs/zh/opencode-bk` 做批量修补；后续通过新生成结果体现导航能力

## 方案概览

采用“页首面包屑 + 分类首页 README”的方案：

- 根目录 `README.md`
  - 保持现有首页职责，继续列出所有顶层页与分类页
  - 不添加页首面包屑
- 分类目录 `README.md`
  - 每个分类目录生成一个 `README.md`
  - 作为该分类的入口页，列出本分类下所有页面
  - 顶部自带面包屑，但它是分类首页生成器直接写出的内容，不走正文页注入逻辑
- 正文页面包屑
  - 所有抓取后生成的正文页在最顶部插入一行面包屑
  - 顶层页显示：`[首页](README.md) / 当前页标题`
  - 分类页显示：`[首页](../README.md) / [分类名](README.md) / 当前页标题`

## 导航示例

### 顶层页面

例如 `docs/zh/opencode/01-简介.md`：

```md
[首页](README.md) / 简介

# 简介
```

### 分类页面

例如 `docs/zh/opencode/使用/07-分享.md`：

```md
[首页](../README.md) / [使用](README.md) / 分享

# 分享
```

### 分类首页

例如 `docs/zh/opencode/使用/README.md`：

```md
[首页](../README.md) / 使用

# 使用

- [Go](01-Go.md)
- [TUI](02-TUI.md)
- [CLI](03-CLI.md)
```

英文目录按相同规则生成，例如 `docs/en/opencode/Usage/README.md` 和正文页顶部的 `Home / Usage / Share` 风格导航，但实际显示文本仍来自当前页面分类名和标题。

例如 `docs/en/opencode/Usage/07-Share.md`：

```md
[Home](../README.md) / [Usage](README.md) / Share

# Share
```

## 模块设计

本次只扩展现有抓取核心 `scripts/scraper_core.py`，不改变入口脚本的职责。

### 新增能力

- 一个面包屑生成函数
  - 输入：当前 `Page`、当前语言配置
  - 输出：Markdown 面包屑文本
- 一个分类首页生成函数
  - 按分类聚合页面并为每个分类写入 `README.md`
- 一个页首导航注入函数
  - 在正文页最终写入前插入面包屑

### 现有函数的改动方向

- `write_page(...)`
  - 只负责写正文页
  - 在写入前先注入页首面包屑
- `generate_index(...)`
  - 只负责生成根目录 `README.md`
  - 不添加页首面包屑
- `scrape_all(...)`
  - 在根目录首页生成后，再为每个分类补生成 `README.md`

职责边界固定如下：

- 根目录 `README.md` 只由 `generate_index(...)` 生成
- 分类目录 `README.md` 只由分类首页生成函数生成
- 正文页只由 `write_page(...)` 写入，并且只在这里做页首面包屑注入

所有这三类文件都采用“整文件重写”的方式生成，不做追加式拼接，避免重复插入导航。

## 链接规则

### 根目录到分类

根目录 `README.md` 中，分类下的页面链接继续保留为：

- `使用/01-Go.md`
- `配置/01-工具.md`
- `开发/01-SDK.md`

本次不额外把根目录首页中的分类标题改成可点击链接，以控制改动面。

### 分类页到首页

分类目录中的 `README.md` 顶部始终回到 `../README.md`。

### 正文页到分类首页

分类正文页中的分类段链接始终指向同目录下的 `README.md`，例如：

- `README.md`

这样可以避免使用 `../README.md#使用` 这类锚点形式，从而规避不同 Markdown 阅读器对标题锚点实现不一致的问题。

## 数据流

单页生成流程变为：

1. 抓取页面并转换为 Markdown
2. 确保有正确的一级标题
3. 做现有的链接本地化和文本后处理
4. 基于页面层级生成面包屑
5. 将面包屑插入到正文最顶部
6. 写入最终 Markdown 文件

单次全量文档生成流程额外增加：

1. 生成根目录 `README.md`
2. 按分类生成分类目录 `README.md`

## 增量与全量行为

- 全量模式
  - 清理当前语言输出目录
  - 重新生成所有正文页、根目录首页和分类首页
- 增量模式
  - 继续保留“已存在正文页跳过”的语义
  - 分类首页与根目录首页仍应基于本次发现到的页面清单重写

这样做的结果是：

- 如果只是新增页面，增量模式下目录页仍能更新导航
- 如果调整正文页模板，例如新增面包屑，已有正文页需要一次全量重建来刷新
- 根目录 `README.md` 与分类目录 `README.md` 都属于生成产物，不保证保留手工修改

## 测试设计

使用现有标准库 `unittest` 测试体系补充以下覆盖：

- 顶层页面包屑生成测试
- 分类页面包屑生成测试
- 分类首页 `README.md` 生成测试
- 页首导航插入位置测试，确保导航位于 `# 标题` 之前

脚本级验证仍以实际运行抓取脚本为准：

- 全量生成中文离线文档并抽查分类页
- 全量生成英文离线文档并抽查分类页

## 风险与控制

- 风险：导航重复插入
  - 控制：正文页、根目录首页、分类首页分别由固定函数整文件生成，不对已有文件做原地拼接式修改
- 风险：分类首页与正文页链接规则不一致
  - 控制：统一由共享函数计算相对路径
- 风险：增量模式下旧页面仍无导航
  - 控制：明确导航模板变更后需要做一次全量重建
- 风险：未来出现多级分类
  - 控制：当前范围只支持与现有 `Page.category` 一致的单层分类结构，多级分类不在本次范围内

## 验收标准

- 任意正文页都能直接返回首页
- 分类下的正文页都能直接返回对应分类首页
- 中英文离线文档均具备相同导航结构
- 中文分类导航不依赖标题锚点
- 全量重建后，生成结果中能看到页首面包屑和分类目录 `README.md`
