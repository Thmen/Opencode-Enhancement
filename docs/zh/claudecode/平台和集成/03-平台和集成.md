# 平台和集成

> 选择在哪里运行 Claude Code 以及连接什么工具。比较 CLI、Desktop、VS Code、JetBrains、Web 以及 Chrome、Slack 和 CI/CD 等集成。

Claude Code 在任何地方运行相同的底层引擎，但每个界面都针对不同的工作方式进行了优化。本页面帮助您为工作流选择合适的平台，并连接您已经使用的工具。

## 在哪里运行 Claude Code

根据您喜欢的工作方式和项目所在位置选择平台。

| 平台                                   | 最适合                                               | 您获得的功能                                                                                                                            |
| :----------------------------------- | :------------------------------------------------ | :-------------------------------------------------------------------------------------------------------------------------------- |
| [CLI](../快速开始/02-快速开始.md)             | 终端工作流、脚本编写、远程服务器                                  | 完整功能集、[Agent SDK](../22-以编程方式运行ClaudeCode.md)、第三方提供商                                                                                         |
| [Desktop](../13-使用ClaudeCodeDesktop.md)            | 视觉审查、并行会话、托管设置                                    | Diff 查看器、应用预览、Pro 和 Max 上的[计算机使用](../13-使用ClaudeCodeDesktop.md#let-claude-use-your-computer)和 [Dispatch](../13-使用ClaudeCodeDesktop.md#sessions-from-dispatch) |
| [VS Code](06-在VSCode中使用ClaudeCode.md)            | 在 VS Code 内工作而无需切换到终端                             | 内联 diff、集成终端、文件上下文                                                                                                                |
| [JetBrains](02-JetBrainsIDEs.md)        | 在 IntelliJ、PyCharm、WebStorm 或其他 JetBrains IDE 内工作 | Diff 查看器、选择共享、终端会话                                                                                                                |
| [Web](../07-ClaudeCodeontheweb.md) | 不需要太多操作的长时间运行任务，或应该在您离线时继续的工作                     | Anthropic 托管云、断开连接后继续运行                                                                                                           |

CLI 是终端原生工作的最完整界面：脚本编写、第三方提供商和 Agent SDK 仅限 CLI。Desktop 和 IDE 扩展为了视觉审查和更紧密的编辑器集成而放弃了一些仅限 CLI 的功能。Web 在 Anthropic 的云中运行，因此任务在您断开连接后继续进行。

您可以在同一项目上混合使用多个界面。配置、项目内存和 MCP 服务器在本地界面之间共享。

## 连接您的工具

集成让 Claude 与代码库外的服务协作。

| 集成                                      | 功能                             | 用途                           |
| :-------------------------------------- | :----------------------------- | :--------------------------- |
| [Chrome](01-在Chrome中使用ClaudeCode（测试版）.md)                 | 使用您登录的会话控制浏览器                  | 测试 Web 应用、填充表单、自动化没有 API 的网站 |
| [GitHub Actions](../19-ClaudeCodeGitHubActions.md) | 在 CI 管道中运行 Claude              | 自动化 PR 审查、问题分类、计划维护          |
| [GitLab CI/CD](../20-ClaudeCodeGitLabCICD.md)     | 与 GitHub Actions 相同，但用于 GitLab | GitLab 上的 CI 驱动自动化           |
| [Code Review](../09-CodeReview.md)       | 自动审查每个 PR                      | 在人工审查前捕获错误                   |
| [Slack](05-Slack中的ClaudeCode.md)                   | 响应频道中的 `@Claude` 提及            | 将错误报告转换为团队聊天中的拉取请求           |

对于此处未列出的集成，[MCP 服务器](../29-通过MCP将ClaudeCode连接到工具.md)和[连接器](../13-使用ClaudeCodeDesktop.md#connect-external-tools)让您连接几乎任何东西：Linear、Notion、Google Drive 或您自己的内部 API。

## 远离终端时工作

Claude Code offers several ways to work when you're not at your terminal. They differ in what triggers the work, where Claude runs, and how much you need to set up.

|                                                | Trigger                                                                                        | Claude runs on                                                                                                   | Setup                                                                                                                                | Best for                                                      |
| :--------------------------------------------- | :--------------------------------------------------------------------------------------------- | :--------------------------------------------------------------------------------------------------------------- | :----------------------------------------------------------------------------------------------------------------------------------- | :------------------------------------------------------------ |
| [Dispatch](../13-使用ClaudeCodeDesktop.md#sessions-from-dispatch) | Message a task from the Claude mobile app                                                      | Your machine (Desktop)                                                                                           | [Pair the mobile app with Desktop](https://support.claude.com/en/articles/13947068)                                                  | Delegating work while you're away, minimal setup              |
| [Remote Control](04-使用RemoteControl从任何设备继续本地会话.md)           | Drive a running session from [claude.ai/code](https://claude.ai/code) or the Claude mobile app | Your machine (CLI or VS Code)                                                                                    | Run `claude remote-control`                                                                                                          | Steering in-progress work from another device                 |
| [Channels](../05-使用channels将事件推送到运行中的会话.md)                       | Push events from a chat app like Telegram or Discord, or your own server                       | Your machine (CLI)                                                                                               | [Install a channel plugin](../05-使用channels将事件推送到运行中的会话.md#quickstart) or [build your own](/en/channels-reference)                                      | Reacting to external events like CI failures or chat messages |
| [Slack](05-Slack中的ClaudeCode.md)                             | Mention `@Claude` in a team channel                                                            | Anthropic cloud                                                                                                  | [Install the Slack app](05-Slack中的ClaudeCode.md#setting-up-claude-code-in-slack) with [Claude Code on the web](../07-ClaudeCodeontheweb.md) enabled | PRs and reviews from team chat                                |
| [Scheduled tasks](../40-按计划运行提示词.md)         | Set a schedule                                                                                 | [CLI](../40-按计划运行提示词.md), [Desktop](../13-使用ClaudeCodeDesktop.md#schedule-recurring-tasks), or [cloud](../52-在网络上安排任务.md) | Pick a frequency                                                                                                                     | Recurring automation like daily reviews                       |

如果您不确定从哪里开始，[安装 CLI](../快速开始/02-快速开始.md) 并在项目目录中运行它。如果您不想使用终端，[Desktop](../14-开始使用桌面应用.md) 为您提供相同的引擎和图形界面。
